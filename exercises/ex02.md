# Ex02 — Criar a primeira tool

## Contexto

Agora vamos transformar o endpoint de ticket em uma capacidade reutilizável. A tool deve esconder detalhes HTTP e entregar um dicionário simples para o futuro agente.

## Arquivo permitido

```text
stack_sentinel/mcp_server/tools.py
```

## Contrato

Implemente:

```python
fetch_ticket_context(ticket_id: str, client: Optional[MockServiceClient] = None) -> dict
```

Em sucesso, retorne:

```python
{
    "ok": True,
    "id": "TCK-101",
    "summary": "...",
    "severity": "high",
    "service": "auth-service",
    "status": "open",
    "build_id": "BLD-203"
}
```

Em falha, retorne:

```python
{
    "ok": False,
    "error": "mensagem curta"
}
```

Regras:

- aceite `client` injetado pelos testes;
- se `client` for `None`, crie um `MockServiceClient`;
- não altere a assinatura;
- não retorne o payload bruto inteiro da API;
- preserve os nomes das chaves.

## Validação

```bash
python run.py test ex02
```
