# Ex01 — Explorar o serviço mockado

## Contexto

Antes de criar tools ou agente, valide que existe um serviço externo simulando sistemas internos. Neste exercício, você só precisa confirmar que a mock API responde ao health check.

## Arquivo permitido

```text
stack_sentinel/clients/mock_service_client.py
```

## Contrato

Implemente:

```python
check_mock_service_health(client: Optional[MockServiceClient] = None) -> bool
```

Regras:

- se `client` for `None`, crie um `MockServiceClient`;
- chame o endpoint definido por `MOCK_ENDPOINTS["health"]`;
- retorne `True` somente quando a resposta tiver `ok == True`;
- retorne `False` para qualquer resposta sem `ok`, com `ok == False` ou erro controlado;
- não lance exceção para o usuário da função.

## Validação

```bash
python run.py test ex01
```
