# Ex13 — Node que consulta build

## Contexto

O fluxo de build segue o mesmo padrão do fluxo de ticket, mas usando `build_id` e a tool de build.

## Arquivo permitido

```text
stack_sentinel/agent/nodes.py
```

## Contrato

Implemente:

```python
fetch_build_node(state: AgentState, mcp_client: MCPClient) -> AgentState
```

Regras:

- use `state["build_id"]` se ele já existir;
- se não existir, extraia o ID de `state["user_input"]` usando o padrão `BLD-000`;
- chame `mcp_client.call_tool(BUILD_TOOL_NAME, {"build_id": build_id})`;
- quando `ok=True`, salve o resultado em `context`, preencha `build_id` e deixe `error` como `None`;
- quando não houver ID ou a tool falhar, preencha `error`.

## Validação

```bash
python run.py test ex13
```
