# Ex14 — Node que usa resource/prompt

## Contexto

Nem toda resposta precisa chamar uma tool. Para documentação e triagem, o agente pode buscar um resource e um prompt MCP.

## Arquivo permitido

```text
stack_sentinel/agent/nodes.py
```

## Contrato

Implemente:

```python
fetch_docs_node(state: AgentState, mcp_client: MCPClient) -> AgentState
```

Regras:

- leia `INCIDENT_RESPONSE_RESOURCE` com `mcp_client.read_resource`;
- obtenha `INCIDENT_TRIAGE_PROMPT` com `mcp_client.get_prompt`;
- passe `user_question` e `available_context` como argumentos do prompt;
- salve em `context` um dicionário com as chaves `resource` e `prompt`;
- em falha, preencha `error`.

## Validação

```bash
python run.py test ex14
```
