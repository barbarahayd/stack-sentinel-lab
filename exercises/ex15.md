# Ex15 — Resposta final

## Contexto

O usuário não quer um JSON cru. Ele quer uma resposta útil, com evidências e próximo passo.

## Arquivo permitido

```text
stack_sentinel/agent/nodes.py
```

## Contrato

Implemente:

```python
final_answer_node(state: AgentState) -> AgentState
```

Regras:

- para ticket, inclua ID, resumo, severidade, serviço, status e próximo passo;
- para build, inclua ID, status, serviço, etapa falha e próximo passo;
- para docs, use o contexto de resource/prompt;
- para `unknown` ou erro, retorne fallback claro;
- a resposta deve conter seções ou marcadores equivalentes a `Resumo`, `Evidências` e `Próximo passo`;
- salve a resposta em `state["final_answer"]`.

## Validação

```bash
python run.py test ex15
```
