# Ex05 — Criar segunda tool

## Contexto

O agente também precisará investigar builds quebrados. Esta tool segue o mesmo padrão da tool de ticket.

## Arquivos permitidos

```text
stack_sentinel/mcp_server/tools.py
stack_sentinel/mcp_server/server.py
```

## Contrato

Implemente:

```python
fetch_build_status(build_id: str, client: Optional[MockServiceClient] = None) -> dict
```

Em sucesso, retorne:

```python
{
    "ok": True,
    "id": "BLD-203",
    "status": "failed",
    "service": "auth-service",
    "branch": "main",
    "failed_step": "integration-tests",
    "log_excerpt": "..."
}
```

Em falha, retorne `ok=False` e `error`.

O registro da tool `BUILD_TOOL_NAME` já existe no scaffolding. Ele deve continuar funcionando.

## Validação

```bash
python run.py test ex05
```
